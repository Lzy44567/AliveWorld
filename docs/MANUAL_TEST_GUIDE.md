# v1.1 活世界 Alpha 人工测试指南

这份指南只要求你按步骤操作，不需要理解代码、JSON 或自动测试。

## 准备一次测试环境

1. 在项目根目录运行：

   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\install_test_assets.ps1 -Force
   ```

   它会在你本机的 `data/` 放入四张名称以“测试_”开头的卡。它们不会进入 Git。

2. 启动 AliveWorld，打开右侧资产库。每个测试都新建一个存档，不要在同一存档连续测试不同资产；旧聊天记录可能仍带着前一张卡的上下文。

3. 测试暗流实体前，进入设置，勾选：`显示名称`、`显示动机`、`显示气泡`。测试编辑时再勾选 `允许编辑`。

## 测试 1：世界书能启用，也能关闭

1. 新建存档 `AW_WB_ON`。
2. 右侧切到“世界法则”→“全局图鉴”，将 `测试_世界书回声` 载入本局。
3. 切换到“本局专属”，确认该卡开关显示“已启用”。
4. 输入：`AW_WB_TRIGGER_20260710`。
5. 预期：正文出现 `AW_WB_READY_20260710`。

关闭验证：新建存档 `AW_WB_OFF`，同样载入该卡后把开关切到“已封存”，再输入同一触发词。预期正文**不应**出现 `AW_WB_READY_20260710`。

## 测试 2：文风卡能启用，也能关闭

1. 新建存档 `AW_STYLE_ON`，载入 `测试_文风回声`。
2. 输入：`AW_STYLE_TRIGGER_20260710`。
3. 预期：正文末尾出现 `AW_STYLE_READY_20260710`。
4. 用新的 `AW_STYLE_OFF` 存档重复，先将该卡封存；预期不出现该标记。

## 测试 3：角色卡能启用，也能关闭

1. 新建存档 `AW_CHAR_ON`，载入 `测试员_角色卡`。
2. 输入：`AW_CHAR_TRIGGER_20260710`。
3. 预期：正文出现 `AW_CHAR_READY_20260710`。
4. 用新的 `AW_CHAR_OFF` 存档重复，先封存该卡；预期不出现该标记。

## 测试 4：实体、世界推演与显示权限

1. 新建存档 `AW_ENTITY_ON`，载入 `测试_暗流回声` 或 `世界推演`。
2. 设置中勾选名称、动机、气泡；输入：`AW_ENTITY_TRIGGER_20260710`。
3. 预期：本局实体卡显示名称与动机；如果 Overseer 本回合采取行动，会出现对应气泡。实体允许返回空动作，所以没有气泡不必立即判定为故障。
4. 将实体切到“已封存”，再输入任意行动。预期：该实体不应在之后的气泡中行动。
5. 依次取消“显示动机”“显示名称”“显示气泡”，检查卡片与气泡同步隐藏；勾选“允许编辑”后，应能打开实体的普通表单编辑状态、计划、机制、触发器与关系。

## 测试 5：基础游戏回归

在任意普通存档中完成一次：发送行动、查看未来候选、重掷、撤回、重试、保存、关闭后重新读取。预期聊天、状态栏和本局资产保持一致。

## 测试 6：三分钟检查用户偏好证据

这个测试只验证确定性的记录链，不要求 AI 在三分钟内准确理解你的长期心理。

1. 载入任意存档，确认局内高级设置中的“自动收集偏好证据”已开启。
2. 完成一次重掷未来、重试或撤回；也可以生成或重新生成一张故事图片。
3. 点击顶部“偏好”，展开“偏好证据时间线”。
4. 预期：立即看到对应的“重掷未来 / 重试回合 / 撤回回合 / 生成图片”记录，并显示“界面行为”和“待分析”。
5. 点击“重新分析证据”。如果偏好模型可用，分析后可能产生一个或多个候选；展开“查看证据与强度”，应看到弱证据及其来源。AI 也允许认为证据不足而不产生候选。
6. 输入 `/偏好 我喜欢慢节奏的角色互动`。预期：不推进故事回合，偏好页立即出现一条已生效的高先验偏好。

单次重掷没有形成“讨厌该角色”的偏好，反而是正确结果；系统必须保留求胜、输出质量或随机运气等替代解释。

## 如果出现问题，请这样反馈

每个问题只需给我五项信息：测试编号、存档名、输入文本、实际看到的结果、截图。若是实体问题，再补充设置中勾选了哪些实体显示项。这样可以直接对应提交和功能边界定位。

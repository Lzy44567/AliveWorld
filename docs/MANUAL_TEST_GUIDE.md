# v1.1 活世界 Alpha 人工测试指南

这份指南只要求你按步骤操作，不需要理解代码、JSON 或自动测试。

## 准备一次测试环境

1. 在项目根目录运行：

   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\install_test_assets.ps1 -Force
   ```

   它会在你本机的 `data/` 放入四张名称以“测试_”开头的卡。它们不会进入 Git。

2. 启动 AliveWorld，打开右侧资产库。每个测试都新建一个存档，不要在同一存档连续测试不同资产；旧聊天记录可能仍带着前一张卡的上下文。

3. 测试暗流实体前，进入设置，勾选：`显示名称`、`显示动机`、`显示气泡`。测试编辑时再勾选 `允许编辑`。

## 测试 1：世界书能启用，也能关闭

1. 新建存档 `AW_WB_ON`。
2. 右侧切到“世界法则”→“全局图鉴”，将 `测试_世界书回声` 载入本局。
3. 切换到“本局专属”，确认该卡开关显示“已启用”。
4. 输入：`AW_WB_TRIGGER_20260710`。
5. 预期：正文出现 `AW_WB_READY_20260710`。

关闭验证：新建存档 `AW_WB_OFF`，同样载入该卡后把开关切到“已封存”，再输入同一触发词。预期正文**不应**出现 `AW_WB_READY_20260710`。

## 测试 2：文风卡能启用，也能关闭

1. 新建存档 `AW_STYLE_ON`，载入 `测试_文风回声`。
2. 输入：`AW_STYLE_TRIGGER_20260710`。
3. 预期：正文末尾出现 `AW_STYLE_READY_20260710`。
4. 用新的 `AW_STYLE_OFF` 存档重复，先将该卡封存；预期不出现该标记。

## 测试 3：角色卡能启用，也能关闭

1. 新建存档 `AW_CHAR_ON`，载入 `测试员_角色卡`。
2. 输入：`AW_CHAR_TRIGGER_20260710`。
3. 预期：正文出现 `AW_CHAR_READY_20260710`。
4. 用新的 `AW_CHAR_OFF` 存档重复，先封存该卡；预期不出现该标记。

## 测试 4：实体、世界推演与显示权限

1. 新建存档 `AW_ENTITY_ON`，载入 `测试_暗流回声` 或 `世界推演`。
2. 设置中勾选名称、动机、气泡；输入：`AW_ENTITY_TRIGGER_20260710`。
3. 预期：本局实体卡显示名称与动机；如果 Overseer 本回合采取行动，会出现对应气泡。实体允许返回空动作，所以没有气泡不必立即判定为故障。
4. 将实体切到“已封存”，再输入任意行动。预期：该实体不应在之后的气泡中行动。
5. 依次取消“显示动机”“显示名称”“显示气泡”，检查卡片与气泡同步隐藏；勾选“允许编辑”后，应能打开实体的普通表单编辑状态、计划、机制、触发器与关系。

## 测试 5：基础游戏回归

在任意普通存档中完成一次：发送行动、查看未来候选、重掷、撤回、重试、保存、关闭后重新读取。预期聊天、状态栏和本局资产保持一致。

## 测试 6：三分钟检查用户偏好证据

这个测试只验证确定性的记录链，不要求 AI 在三分钟内准确理解你的长期心理。

1. 载入任意存档，确认局内高级设置中的“自动收集偏好证据”已开启。
2. 完成一次重掷未来、重试或撤回；也可以生成或重新生成一张故事图片。
3. 点击顶部“偏好”，展开“偏好证据时间线”。
4. 预期：立即看到对应的“重掷未来 / 重试回合 / 撤回回合 / 生成图片”记录，并显示“界面行为”和“待分析”。
5. 点击“重新分析证据”。如果偏好模型可用，分析后可能产生一个或多个候选；展开“查看证据与强度”，应看到弱证据及其来源。AI 也允许认为证据不足而不产生候选。
6. 输入 `/偏好 我喜欢慢节奏的角色互动`。预期：不推进故事回合，偏好页立即出现一条已生效的高先验偏好。

单次重掷没有形成“讨厌该角色”的偏好，反而是正确结果；系统必须保留求胜、输出质量或随机运气等替代解释。

## 测试 7：用户偏好工坊完整闭环

1. 点击顶部“偏好”，再点击“进入偏好工坊”。预期打开全屏工坊，关闭后再次进入能恢复对话和未发布草稿。
2. 保持“先讨论方案”，输入：`我喜欢慢节奏，但不喜欢几回合都没有任何关系进展。帮我区分这两件事，先不要修改。`
3. 预期：玩家消息立即显示；等待提示明确；AI 回复包含理解、替代解释或设计取舍。右侧草稿不应被直接改变。
4. 如果出现“AI 拟议修改”，点击“提交方案”。低风险措辞修改进入右侧草稿；建议启用或删除时进入黄色确认区，不应直接执行。
5. 右侧点击“新增”，创建一条候选；再编辑其分类和表述。预期操作只改变工坊草稿。
6. 关闭工坊并打开普通偏好卡。发布前，正式偏好卡不应出现草稿修改。
7. 重新进入工坊，确认草稿恢复；点击“撤销草稿修改”，预期最近一次草稿操作恢复。
8. 处理所有黄色待确认项，点击“发布到偏好卡”。关闭工坊后，正式偏好卡应显示发布结果，证据时间线仍完整存在。
9. 在工坊未发布时继续玩一回合或执行“重新分析证据”，再回到工坊。预期提示已安全合并到最新版，草稿修改与新学习结果同时保留；若确实冲突则明确拒绝发布，不能静默覆盖。

## 测试 8：总工坊与角色/文风/实体协作

1. 点击顶部“工坊”。预期在同一入口看到世界书、角色卡、文风卡、实体卡和偏好卡五类，而不是分别寻找入口。
2. 切换“全局资产 / 本局专属”，再切换不同资产类型。预期列表与当前范围一致；没有载入存档时，本局按钮不可用。
3. 选择一张角色卡进入工坊，保持“先讨论方案”，输入：`先分析这个角色哪里空洞或俗套，不要修改。` 预期消息先显示，AI 只讨论；正式角色卡和右侧草稿均不应被直接覆盖。
4. 要求 AI 给出一版具体人物修改。审阅蓝色提案后点击“提交方案”，再在右侧手动微调，点击“保存到草稿”。关闭并重新进入，预期对话和草稿恢复。
5. 点击“发布”，再打开普通角色编辑器。预期人物设定与登场情境更新，已有立绘仍存在；AI 不应擅自改变“玩家扮演角色”。
6. 对文风卡重复上述流程，要求：`把这张卡整理成可执行的视角、节奏、句式、描写重点和禁忌。` 发布后普通编辑器应看到新的文风正文。
7. 对实体卡重复流程，要求区分“机制、计划、近期行动”。预期右侧使用普通输入框和列表，不要求玩家编辑 JSON；发布后实体运行开关和已有因果引用不应被 AI 改变。
8. 工坊草稿未发布时，在普通编辑器修改同一资产并保存，再尝试发布旧草稿。预期明确拒绝覆盖，提示关闭后重新进入。
9. 进入“设置 → API 配置”。预期正文 API、偏好分析模型和折叠的“记忆压缩模型”位于同一分类；左侧不再有单独“记忆模型”栏目。

## 测试 9：游戏/工坊模式与三栏复用

1. 启动后观察顶部中央。预期显示“游戏 / 工坊”分段开关，而不是单独的工坊按钮。
2. 在游戏模式载入一个存档并记住当前正文、状态栏和右侧标签，然后切换到工坊。预期主界面原地变为三栏工坊，不弹出覆盖全屏的大窗口。
3. 预期左侧显示“工坊草稿”，中央显示协作对话和输入框，右侧显示世界书、角色、文风、实体、偏好五类资产。
4. 在右侧选择一张世界书。预期左侧出现概述、公理和条目，中央进入该世界书的既有工坊会话；可以在左侧新增/编辑/启停条目，删除会进入中央待确认区。
5. 切换到角色、文风和实体，分别选择资产。预期左侧表单结构随资产类型变化，而中央始终复用同一套讨论、提案与发布流程。
6. 切换到偏好。预期左侧显示偏好条目及后验摘要，可以新增、编辑、启停或申请删除；证据和后验不能被工坊 AI 直接编辑。
7. 在任意工坊留下未发布草稿，切换回游戏。预期原来的存档、正文、状态栏和资产栏立即恢复；发送故事行动不受工坊影响。
8. 再切换到工坊。预期刚才的资产选择、对话和未发布草稿仍保留，不需要重新从总入口选择。
9. 从普通资产卡或资产编辑器点击“进入工坊”。预期关闭当前编辑弹窗并切换到主工坊模式，直接选中对应资产。

## 如果出现问题，请这样反馈

每个问题只需给我五项信息：测试编号、存档名、输入文本、实际看到的结果、截图。若是实体问题，再补充设置中勾选了哪些实体显示项。这样可以直接对应提交和功能边界定位。

# 总工坊模式

## 目标

总工坊为世界书、角色卡、文风卡、暗流实体和用户偏好卡提供一个统一入口。它不是把所有资产塞进同一套 JSON 表单，而是统一以下协作骨架：

1. 选择全局资产或当前存档副本。
2. 通过连续对话完成“理解目标 → 讨论取舍 → 提出方案 → 审阅 → 修改草稿”。
3. 草稿与正式资产隔离并自动保存。
4. 玩家可直接修改结构化草稿、撤销和发布。
5. 工坊不推进故事回合、世界时间或 Overseer。

## 当前实现

- 顶部“工坊”进入统一资产选择页，可切换世界书、角色、文风、实体和偏好。
- 世界书继续使用专用条目、公理和概述事务协议。
- 偏好继续使用专用证据、候选、Python 后验和高风险确认协议。
- 角色、文风、实体共享资产工坊事务内核，但使用不同提示词职责与字段白名单。
- 角色工坊处理人物设定与登场情境，不允许 AI 改玩家扮演身份或立绘引用。
- 文风工坊处理可执行的叙事规范，不写具体世界事实。
- 实体工坊处理动机、状态、机制、计划、近期行动、触发器和关系；不允许 AI 改运行开关、因果引用或资产名称。
- 发布前若原资产已在普通编辑器或运行时发生变化，工坊拒绝覆盖并要求重新进入。
- 模板只能克隆后发布，不能被工坊直接覆盖。

## 信息架构判断

当前采用顶部统一入口和全屏工坊，而不是立即把整个正文三栏界面改造成永久“游戏/工坊”路由。共同壳层已经成立，但各类草稿结构差异很大；先验证统一入口、返回路径和五类真实闭环，再决定是否让工坊永久占据正文工作区。

聊天模式仍是独立玩法。它需要角色扮演协议、短消息节奏、关系记忆、群聊与自拍语义，不能因为已有总工坊就作为空白标签加入。

## 后续扩展

- 角色工坊：从故事正文和分级记忆回溯提取普通 NPC，经玩家确认形成角色卡。
- 角色视觉档案：参考图、稳定外观描述和全局立绘共同管理。
- 文风工坊：示例片段对照、负面规范和模型适配测试。
- 实体工坊：与当前故事因果账本进行受控的来源关联审阅。
- 统一工坊导入文件、版本对比和分支克隆。

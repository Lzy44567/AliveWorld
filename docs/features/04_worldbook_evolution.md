# 04 世界书：检索、工坊、拓展与演化

## 职责

- 世界书拓展：增加此前没有覆盖的新主题和条目。
- 世界书演化：从既有公理、规则和临时创作偏好推导体系后果。
- 世界演化：故事时间中的势力与环境变化，属于实体、世界推演和暗流因果账本。

## 已确认方向

- 全局世界书载入后复制为局内副本；当前代码已具备物理复制与另存全局能力。
- 世界书以向量语义召回为主要方向，关键词作为强触发和无向量降级。
- 小中规模先使用持久化向量缓存与内存相似度检索，不强制引入重型向量数据库。
- 本地嵌入模型按需下载；未安装或运行失败时不得阻塞正文。
- 系统标签与自由标签并存。系统标签对玩家可见、可补全、可修改，并提供 `?` 说明。
- 系统标签首版包含：绝对规则、常驻、传闻、玩家设定、AI推断、Overseer推断、待确认、待删除。
- 工坊从世界书编辑页进入，使用独立聊天会话和草稿；AI通过受限JSON操作修改草稿，后端校验并保存快照。
- 工坊提供“从一句话创建、拓展新领域、演化已有设定”三个入口，并支持分批整本演化。
- AI可自动修改草稿，最终由玩家发布；新公理、绝对规则、冲突覆盖和删除必须单独确认。
- 工坊默认采用“先讨论方案”：AI先说明目标理解、与既有设定的关系、推导依据和取舍，拟议操作独立保存且不改变草稿。玩家可提交方案，或主动切换“允许 AI 修改草稿”；任何模式下公理、绝对规则和删除仍需逐项确认并展示完整内容。
- 工坊协作按探索、设计、提案、写入四个阶段连续推进。讨论和比较方案的轮次允许零操作，不能为了显得有产出而强行新增条目。

## 当前实现

- `core/worldbook.py`：条目规范化、稳定ID、系统标签、检索结果和语义提供器边界。
- 当前检索实现关键词、常驻、启用/待确认过滤和字符预算；向量提供器尚未绑定。
- 世界书编辑器支持资产和条目标签，后端保存时再次规范化。
- 日志记录实际命中原因和预算裁剪。
- 本地多语言嵌入模型支持按需后台下载、启停、持久化向量缓存和异常降级；不下载时维持关键词模式。
- 模型管理显示来源、大小、落盘位置和进度，支持停止并保留断点、继续与卸载；下载进程可独立终止。
- `core/worldbook_workshop.py` 提供独立事务草稿、快照、撤销、待确认操作和发布。
- 工坊API、AI受限操作适配器和独立聊天界面已经接通；支持三种模式、建议按钮、高风险确认、撤销和发布。
- 建议按钮只填入输入框，不直接发送；公理编辑和确认使用有序预览，但底层存储不携带序号，避免重排后双重编号。
- 工坊未发布草稿可恢复并在书间切换；右侧支持手动编辑概述、公理和条目，AI可按稳定条目ID修改指定内容。
- 工坊会话记录源世界书版本。编辑页若已更新原文件，陈旧工坊草稿不得遮蔽最新版；系统从最新版创建新会话并保留旧草稿文件。相同内容的自动保存不制造脏草稿。
- 世界公理保持“一条一项”的数组语义；玩家界面提供编号预览和逐条“让 AI 讨论修改”，序号只是展示层，不写入公理正文。
- 局内新设定支持回合后异步捕获和局内开关；首版只新增稳定低风险条目，高风险/确认模式条目标记待确认并排除检索。
- 正文结算 JSON 先返回 `worldbook_capture_needed`。只有严格布尔值 `true` 才允许启动独立捕获 AI；字段缺失、字符串或 `false` 都跳过，避免每回合固定消耗第二次请求并诱导捕获器凑内容。
- 主书/参考书身份未实现前，捕获目标暂为第一本启用的局内世界书。
- 世界书所有关键写入采用原子替换；编辑器可直接接受并启用或拒绝删除待确认候选。

# 03 动态未来候选

## 当前决策

- 现有游戏本来就是单层候选。本阶段优化该核心闭环，不预生成多层 `N×n` 剧情树。
- 每回合根据世界书、角色状态、近期事实、实体与暗流因果生成若干近期未来。
- 先判断候选是否符合硬性事实，再为合格候选分配非负相对权重；权重无需合计 100。
- `eligible=false` 的候选权重归零，不进入 Python 随机池，但可保留在调试展示与日志中。
- `basis` 保存关键事实依据，用于调试模型判断；它不是客观概率证明。
- 候选必须实质不同，但不按改善、恶化、中立等主角中心类别硬凑。
- 重掷与正常回合复用同一候选领域逻辑。

## 当前实现边界

`core/future_candidates.py` 是候选清洗、资格过滤、相对权重与抽取的唯一领域入口。旧协议缺少 `eligible` 或 `basis` 时默认兼容。

本切片不做因素逐项计分、概率连乘、通用条件表达式或世界书演化。后续先依据真实日志验证候选事实性，再专项设计世界书的检索、冲突识别、演化草稿与玩家确认边界。

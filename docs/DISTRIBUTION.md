# AliveWorld 发行与打包方向

## 不应提交到Git的内容

- `.venv/`、`node_modules/`：体积大、平台相关、不可复现。
- `config.yml`：包含私人API凭据。
- `data/`中的个人资产、存档、工坊草稿、模型和缓存。
- 运行日志。

## 源码试玩版

源码版使用 `install_windows.bat` 在项目内创建环境。它适合开发者和早期测试者，但仍要求玩家预先安装Python和Node。

## GitHub Release便携版

正式便携版应在CI或干净构建机上生成，不把运行环境直接提交进Git：

1. 前端执行生产构建，输出静态文件。
2. FastAPI后端负责同时托管API和前端静态文件。
3. 使用Python应用打包工具封装解释器、后端代码和依赖。
4. 生成单独的用户数据目录，程序升级不得覆盖存档和资产。
5. 输出ZIP或安装程序，并附带校验值和版本说明。

## Steam/商业桌面版

Steam版需要在便携版基础上增加：

- 桌面窗口壳和单实例管理。
- Windows代码签名、崩溃日志和自动更新策略。
- 将存档放入稳定的用户数据目录，而不是安装目录。
- API Key安全存储与隐私说明。
- 可选本地模型的下载许可、来源、卸载和磁盘空间管理。
- 第三方模型、依赖、字体和素材的商业许可证审查。

正式打包是独立发行阶段，不应通过把当前开发机的Python或Node目录复制进仓库来完成。

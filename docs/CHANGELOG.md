# 更新日志

本日志按可验证的开发切片记录。每条完成项对应一个独立 Git 提交，提交说明包含版本标签和作者/环境标签。

## [v1.1.0] - 2026-07-10 - 切片 2：Overseer 协议升级

作者/环境：`codex-gpt5.6Terra`

### 变更

- Overseer 现在明确可以返回空动作，不再要求每回合制造暗流事件。
- 默认统筹者不再称为“世界意志”；“天道/世界意志”只在世界观真实存在时作为独立实体。
- 事件协议支持状态、增量计划、持续机制、未来触发线索、关系变化和主线可用线索。
- 模型只接收活跃实体；即使返回封存或不存在实体的动作、更新、删除，也会被后端忽略。
- 影子账本保存结构化依据，并仅投影有限线索给主线上下文。

### 验证

- `system_prompts.yml` YAML 解析通过。
- `python -m unittest discover -s tests -v`：4 项通过，包含封存实体拒绝执行的行为测试。
- `npm run build` 通过。

### 已知范围

- 尚未提供“世界推演”实体模板或 UI 可见性选择；这是后续切片，不影响本协议使用现有实体卡。

## [v1.1.0] - 2026-07-10 - 切片 1：实体领域边界

作者/环境：`codex-gpt5.6Terra`

### 新增

- 实体领域模型：支持动机、状态、近期行动、计划、机制、触发、关系、重要性与启用状态。
- 本局实体 YAML 仓储，集中负责读取和同步。
- 结构化影子账本，并提供有长度限制的主线注入摘要。
- 不联网的领域与 Overseer 行为测试。
- v1.1.0 路线图、设计总纲和决策池。

### 兼容与修复

- 兼容 v1.0.0 的实体 `goal` 字段和字符串影子账本。
- `is_active=false` 的实体会保留在本局实体库，但不进入 Overseer 上下文。
- 会话层不再直接承担实体 YAML 读写职责。

### 验证

- 全量 Python 静态语法检查通过。
- `python -m unittest discover -s tests -v`：4 项通过。
- `npm run build` 通过。

### 已知范围

- Overseer 仍使用 v1.0.0 输出协议；下一切片将升级其动作、状态、计划、机制、触发线索和关系变化协议。
- 当前未改 UI、未自动注入“世界推演”、未启动服务或调用真实模型。

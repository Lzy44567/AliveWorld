## [v1.3.0] - 2026-07-12 - 切片 1：动态未来候选可信化

作者/环境：`codex-gpt5.6sol`

### 新增

- 新增独立 `future_candidates` 领域模块，统一候选清洗、资格过滤、相对权重校验、重掷排除与 Python 随机抽取。
- 动态未来协议增加 `eligible` 与 `basis`：硬条件不满足的候选保留在调试信息中，但权重归零且不进入随机池。
- 日志记录每个候选的事实依据、相对权重，以及选中项归一化后的实际抽取概率。

### 调整

- 权重不再要求合计 100，玩家端改为显示“相对权重”，不再错误追加百分号。
- 提示词要求先符合世界事实，再追求实质差异；不强制生成有利、恶化、中立等固定类别。
- 旧存档和旧模型返回若缺少新字段，仍按默认有资格候选兼容处理。

### 暂不实现

- 因素逐项计分、概率连乘、多层 `N×n` 展开与世界书自动演化不进入本切片。

## [v1.2.0] - 2026-07-11 - 切片 1：暗流因果账本闭环

作者/环境：`codex-gpt5.6sol`

### 新增

- 新增独立暗流因果账本领域模型，支持一次性、持续性、可演化影响，多来源链接和唯一 ID。
- Python 自动维护存在回合、判断次数、触发次数与四种消费规则。
- 实体卡同步保存影响索引；中心账本是完整内容的唯一真源。
- 来源实体死亡支持 `remove / release / keep`：雇佣关系可取消、死亡瘟疫可在下回合强制释放、环境和社会影响可继续存在。
- 变数 AI 返回条件判断和依据；Python 校验活跃 ID，所有满足条件的影响交给正文 AI 兑现。
- 正文 JSON 新增 `resolved_influences`，本地日志记录完整条件判断和兑现链。
- 新增默认关闭的暗流影响触发气泡、完整账本调试开关，以及当前故事账本的创建、编辑和取消接口/UI。
- 局内实体卡显示同 ID 的关联影响索引，可跳转到中心账本。

### 暂不实现

- 概率投掷、寿命/威力衰减、世界时间计算与逐条 AI 演化仍为 P2。

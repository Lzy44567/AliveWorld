## [v2.2.0-beta] - 2026-06-01

### 🚀 新增功能
- **局内设定双向流通**：实装了局内资产向全局图鉴的“另存为 / 覆盖推送”机制，并允许在局内面板中重命名资产卡片。
- **实体库沙盒化同步 (Entity Sync)**：彻底移除了底层的硬编码测试实体。现在，大模型 Overseer 推演出的实体动态（诞生、转变、湮灭）将实时同步并读写到该存档的 `entities` 物理文件夹中！
- **设置面板扩充与持久化**：全面扩充了设置中的功能占位符（包含时间显示、实体调试、记忆压缩等），并接入了 `localStorage` 持久化，刷新不再丢失配置。

### 🔧 缺陷修复
- **修复：幽灵实体问题**：解决了即使玩家实体库只有“天道”，游戏内依然会出现“潜伏的危机”等幽灵暗流的问题。
- **优化：角色化身判定**：在角色卡编辑中加入了 `is_player` 判定开关。


## [v2.3.0-beta] - 2026-06-XX

### 🚀 新增功能
- **局内编辑与全局推送闭环**：在侧边栏局内专属面板中，实现了微调、启用/封存开关，以及“推送全局（另存为）”的完整交互。
- **极致创世体验**：重构了创世弹窗，改为“仅输入时间线名称与宇宙导向”的极简模式，降低开局心智负担。
- **前端系统设置持久化**：补充了推演调试、世界时间显示、API配置等 UI 占位，并接入 `localStorage` 实现刷新不丢失。
- **AI环境降临初始化**：在全新故事线中加入了“让AI降临并初始化世界状态”的功能按钮，由大模型自动生成进度条与场景属性。

### 🔧 遗留已知问题 (待下个版本修复)
- 前端的 API 设置尚未与后端的 `config.yml` 真实打通。
- 实体库中 AI 动态创建新实体的落地成功率较低。
- 聊天气泡中“暗流涌动”的显示错误地绑定到了“掷骰结果”开关上。
